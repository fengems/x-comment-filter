# TCFilter 扩展 UI 设计图

> 优化目标：Popup 做成"快速概览+状态感知"，Options 做成"侧边导航+模块化配置"。

---

## 一、问题诊断（现状）

### Popup 现状
```
┌─────────────────────────────┐
│ 🛡️ TCFilter          [开关] │   ← 头部 OK
├─────────────────────────────┤
│        0                    │
│      已屏蔽                  │   ← 单薄，无分类、无今日数据
├─────────────────────────────┤
│ 过滤范围    [仅评论区 ▾]     │
│ 屏蔽方式    [折叠 ▾]         │   ← 设置项平铺，视觉权重 = 统计
│ ☑ 同时过滤用户名             │
│ ☐ 试运行                     │
│ ☐ 调试日志                   │
├─────────────────────────────┤
│ [🔄 同步词库] [⚙️ 高级设置]  │
├─────────────────────────────┤
│        最新                  │   ← 同步状态无视觉差异
└─────────────────────────────┘
```
**问题：** ① 无"当前页面"上下文 ② 统计单薄 ③ 三类信息权重相同 ④ 调试日志这种低频项占位

### Options 现状
6 个 section 纯垂直堆叠，需要长滚动；AI 设置（复杂）和候选确认（高频）都在底部，难找。

---

## 二、设计原则

1. **Popup = 瞬态概览**：打开即懂、3 秒决策、只放高频项，低频项进 Options
2. **Options = 模块化配置**：左侧导航分类，告别长滚动，每个 tab 聚焦一个主题
3. **状态可视化**：让用户"看见"过滤在发生（今日数据、当前页面状态、规则数）
4. **视觉层级**：主要 > 次要 > 罕用，用大小/颜色/折叠区分

- **同步时间**：比"最新"更具体

## 六、交互优化清单

| 位置 | 优化点 |
|---|---|
| Popup 统计 | 单数字 → 双卡（总/今日） |
| Popup 状态条 | 新增"正在保护·本页过滤 N 条 XX"（需 content script 上报当前页计数） |
| Popup 折叠 | 试运行/调试默认收起到"高级"折叠组 |
| Options 导航 | 垂直堆叠 → 左侧 5 tab 导航 |
| Options 规则 | 加分类筛选 tab（全部/色情/博彩/...）+ 来源统计 |
| Options AI | 候选确认置顶（带数量红点），后端设置置底 |
| Options 统计 | 加进度条可视化类别分布 |
| Options 数据 | 历史记录可视化（表格而非隐藏 modal） |
| 全局 | 导航 tab 显示待处理数量（如 AI 候选 3 的红点） |

## 七、新增数据需求

实现 Popup 状态条需要 content script 上报"当前页面过滤计数"。方案：
- content script 维护一个 `pageBlockedCount`，过滤时累加，SPA 切页清零
- 通过 `chrome.storage.session`（MV3 会话级存储）或消息上报给 popup
- popup 打开时读取并展示

## 八、实现优先级

1. **P0（立刻做）**：Options 侧边导航改造（解决"找不到东西"的核心痛点）
2. **P0**：Popup 双卡统计 + 低频项折叠（低成本高收益）
3. **P1**：Options AI 候选置顶 + 数量提示
4. **P1**：统计进度条可视化
5. **P2**：Popup 当前页面状态条（需新增 content→popup 数据通道）
6. **P2**：规则分类筛选、历史表格化
