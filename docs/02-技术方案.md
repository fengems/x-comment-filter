# 推特/X 评论过滤扩展 · 技术方案

> 基于 `01-调研报告.md` 的结论设计。
> 一句话定位：**本地优先 + 折叠 UI + 多维规则引擎 + 可选 LLM 精判**的推特评论垃圾信息过滤扩展，主打色情 bot / 博彩广告 / 引流话术。

---

## 一、产品定位与差异化

### 1.1 核心问题
推特评论区充斥色情广告、博彩广告、引流机器人，严重污染阅读体验。现有扩展要么规则太粗（误伤）、要么直接删除不可恢复、要么依赖云端有隐私顾虑。

### 1.2 差异化（调研发现的市场空白，全做）

| 维度 | 现有产品 | 咱们 |
|---|---|---|
| 屏蔽交互 | 多数直接 display:none | **折叠占位卡**（"已屏蔽 N 条，[展开]"），可批量展开/还原 |
| 规则模型 | 扁平（所有词共享全局开关） | **规则对象**：每条带 type/scope/mode/白名单 |
| 判定方式 | 纯关键词 或 纯云端 | **本地关键词预筛 + 可选 LLM 精判**（混合） |
| 误伤防护 | 缺失 | **词边界 + 白名单 + dry-run + 可撤销** |
| 词库更新 | 静态 或 复杂共建 | **GitHub CDN + ETag 增量**（轻量、零后端、可 fork） |
| 隐私 | 云端方案上传文本 | **默认零网络**，LLM 为可选 opt-in |

### 1.3 MVP 范围（先做这些）
1. 评论/Feed 双场景过滤（可开关）
2. 折叠占位卡 UI
3. 多维规则引擎（关键词、用户名、正则、特殊字符）
4. 本地词库 + GitHub 词库同步
5. Popup 设置 + 统计 + 历史
6. 白名单 + 误伤防护

### 1.4 后续 Roadmap（MVP 之后）
- LLM 精判（opt-in，用户填 key 或走自建 Worker）
- 模拟点击 mute/block（带限速队列）
- 社区公榜（参考 MXGA Bloom filter 方案）
- KOL 评分 / 用户摘要（参考 MXGA 未实现部分，做咱们的特色）

## 二、技术栈选型

| 层 | 选型 | 理由 |
|---|---|---|
| Manifest | **MV3** | 趋势，Chrome 力推，MV2 即将下架 |
| 构建 | **WXT** | 现代化扩展框架，自动生成 manifest、HMR、支持多浏览器；MXGA/twitter-blocker 都用 |
| UI（popup/options） | **React + Tailwind** | 生态成熟，WXT 原生支持 |
| Content Script 运行时 | **vanilla DOM + Shadow DOM** | 性能敏感，不引入运行时框架（参考 MXGA：popup 用 React，content 用 vanilla） |
| 语言 | **TypeScript** | 类型安全（避开 xModerator 无类型的坑） |
| 状态存储 | `chrome.storage.local`（数据）+ `sync`（设置，注意 100KB 配额） | 参考双通道设计 |
| 测试 | **Vitest**（纯逻辑）+ 手动 | 至少给规则引擎、DOM 提取、反绕过清洗写单测 |

**注入方式决策**：Content script 跑在 **isolated world**（默认）。
- 优势：合规、不碰页面 JS 上下文、不易被 X 反检测、调试简单
- 代价：读不到 React 内部 state
- 折中：MVP 用 isolated world + DOM 提取（够用）；若未来要做 KOL 评分需读 profile fiber 数据，再评估 main world 注入（参考 Control Panel for Twitter / MXGA 的 fiber 直读）

## 四、核心技术设计（重点）

### 4.1 DOM 监听（ObserverManager）

综合 x-comment-blocker 的批处理 + MXGA 的防抖兜底：

```ts
// lib/observer.ts
export class ObserverManager {
  private observer: MutationObserver;
  private pending = new Set<HTMLElement>();   // 批处理缓冲
  private debounceTimer: ReturnType<typeof setTimeout> | null = null;
  private lastUrl = location.href;

  constructor(private onBatch: (nodes: HTMLElement[]) => void) {
    this.observer = new MutationObserver(this.handle);
  }

  start() {
    this.observer.observe(document.body, {
      childList: true,
      subtree: true,
      // 不配 characterData，避免文本抖动
    });
    // 兜底定时重扫（防虚拟列表边缘情况）
    setInterval(() => this.flush(), 4000);
  }

  private handle = (mutations: MutationRecord[]) => {
    if (!chrome.runtime?.id) { this.observer.disconnect(); return; }  // 扩展已卸载

    // SPA 路由切换 → 清缓存
    if (location.href !== this.lastUrl) {
      this.lastUrl = location.href;
      cacheManager.clear();
    }

    for (const m of mutations) {
      for (const node of m.addedNodes) {
        if (node.nodeType !== Node.ELEMENT_NODE) continue;
        this.collectTweets(node as HTMLElement);
      }
      // 关键：处理虚拟列表复用（X 复用 cellInnerDiv 只换内容）
      if (m.target.nodeType === Node.ELEMENT_NODE) {
        const reused = (m.target as HTMLElement)
          .closest?.('[data-testid="cellInnerDiv"]');
        if (reused) this.pending.add(reused as HTMLElement);
      }
    }

    this.scheduleFlush();
  };

  private collectTweets(node: HTMLElement) {
    if (node.getAttribute?.('data-testid') === 'cellInnerDiv') {
      this.pending.add(node);
    } else {
      node.querySelectorAll?.('[data-testid="cellInnerDiv"]')
        .forEach(t => this.pending.add(t as HTMLElement));
    }
  }

  private scheduleFlush() {
    if (this.debounceTimer) return;
    this.debounceTimer = setTimeout(() => this.flush(), 100);  // 100ms 防抖
  }

  private flush() {
    this.debounceTimer = null;
    if (this.pending.size === 0) return;
    const batch = Array.from(this.pending);
    this.pending.clear();
    this.onBatch(batch);   // 一次性处理
  }
}
```

**设计要点**：
- `childList + subtree`，不配 characterData
- `pending` Set 批处理，100ms 防抖合并
- 虚拟列表复用回捞（x-comment-blocker 踩坑经验）
- `chrome.runtime?.id` 卸载保护
- 4s 兜底定时（MXGA 经验）
- URL 变化清缓存

### 4.2 节点定位 + 文本提取（DomExtractor）

**多 selector 兜底链**（参考 twitter-blocker）：

```ts
// lib/dom-extractor.ts
const TWEET_CELL = '[data-testid="cellInnerDiv"]';
const TWEET_ROOT_FALLBACKS = ['[data-testid="tweet"]', 'article', '[role="article"]'];
const TEXT_SELECTORS = [
  '[data-testid="tweetText"]',           // 当前最稳锚点
  'div[dir="auto"] [data-testid="tweetText"]',
  '[lang] [data-testid="tweetText"]',
  'div[lang]',
  'span[dir="auto"]',
];

export function extractTweet(cell: HTMLElement): TweetData | null {
  const root = TWEIT_ROOT_FALLBACKS.map(s => cell.querySelector(s)).find(Boolean)
    ?? cell;
  const text = extractText(root);
  const username = extractUsername(cell);
  const handle = extractHandle(cell);
  if (!text && !handle) return null;
  return { cell, root, text, username, handle,
           hasMedia: !!cell.querySelector('[data-testid="tweetPhoto"], [data-testid="videoPlayer"]'),
           isPromoted: !!cell.closest('[data-testid="placementTracking"]') };
}

// 把 <img alt>(emoji) 也算进文本 —— x-comment-blocker 经验
function extractText(node: Element): string {
  let text = '';
  const walk = (n: Node) => {
    if (n.nodeType === Node.TEXT_NODE) { text += n.textContent; return; }
    if (n.nodeType !== Node.ELEMENT_NODE) return;
    const el = n as Element;
    if (el.tagName === 'IMG' && el.alt) { text += el.alt; return; }
    el.childNodes.forEach(walk);
  };
  walk(node);
  return text;
}
```

### 4.3 评论区判定（ContextDetector）—— 咱们的核心语义

**只过滤评论，不误伤主推文**（x-comment-blocker 的精华三件套）：

```ts
// lib/context.ts
export type PageContext = 'feed' | 'status' | 'profile' | 'other';

export function getPageContext(): PageContext {
  const path = location.pathname;
  if (/\/status\/\d+/i.test(path)) return 'status';      // 推文详情页(评论区)
  if (/^\/[^/]+$/.test(path) || /^\/[^/]+\/(with_replies|likes)$/i.test(path))
    return 'profile';
  if (path === '/' || path === '/home' || path === '/search') return 'feed';
  return 'other';
}

// 判断某条是不是"主推文本身"（永远放行）—— 靠数据 id 比对，不靠位置
export function isMainTweet(cell: HTMLElement): boolean {
  const statusMatch = location.pathname.match(/\/status\/(\d+)/i);
  if (!statusMatch) return false;
  const pageStatusId = statusMatch[1];
  const timeNodes = cell.querySelectorAll('time');
  for (const t of timeNodes) {
    const link = t.closest('a');
    const m = link?.getAttribute('href')?.match(/\/status\/(\d+)/i);
    if (m) return m[1] === pageStatusId;   // id 相同 = 主推文
  }
  return false;
}

// 决定是否处理该节点
export function shouldProcess(cell: HTMLElement, settings: Settings): boolean {
  const ctx = getPageContext();
  const main = isMainTweet(cell);
  // 只过滤评论模式：仅 status 页 + 非主推文
  if (settings.scope === 'comments' && !(ctx === 'status' && !main)) return false;
  // 推广推文单独处理（已有 X 原生标记）
  return true;
}
```

### 4.4 规则引擎（RuleEngine）—— 咱们的差异化核心

**规则对象模型**（避开 xModerator/x-comment-blocker 的扁平规则）：

```ts
// lib/rules/types.ts
export type RuleType = 'keyword' | 'regex' | 'username' | 'specialChars' | 'emoji' | 'link';
export type RuleScope = 'comment' | 'feed' | 'all';   // 这条规则作用于哪个场景
export type RuleMatch = 'exact' | 'wordBoundary' | 'substring';  // 匹配精度
export type RuleSource = 'builtin' | 'user' | 'cloud';

export interface Rule {
  id: string;
  type: RuleType;
  value: string;          // 关键词 / 正则源 / 用户名
  match: RuleMatch;       // 默认 wordBoundary（防误伤，xModerator 的教训）
  scope: RuleScope;
  enabled: boolean;
  source: RuleSource;
  whitelist?: string[];   // 例外词（命中主词但含例外词则放行）
  category?: 'porn' | 'gambling' | 'promo' | 'spam' | 'custom';
  createdAt: number;
}
```

**匹配引擎（重点：词边界 + 反绕过）**：

```ts
// lib/rules/engine.ts
import { sanitize } from './sanitize';   // 反绕过清洗

export class RuleEngine {
  private compiled: CompiledRule[] = [];

  update(rules: Rule[]) {
    this.compiled = rules.filter(r => r.enabled).map(this.compile);
  }

  private compile(rule: Rule): CompiledRule {
    let regex: RegExp;
    if (rule.type === 'regex') {
      regex = new RegExp(rule.value, 'i');
    } else {
      // 关键词：根据 match 精度构造
      const esc = rule.value.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
      const boundary = rule.match === 'wordBoundary';
      // 中文用 (?<!\w) (?!\w) 模拟词边界（\b 对 CJK 无效）
      const cjkBoundary = rule.match !== 'substring';
      regex = new RegExp(
        cjkBoundary ? `(?<![\\p{L}\\p{N}])${esc}(?![\\p{L}\\p{N}])`
                    : (boundary ? `\\b${esc}\\b` : esc),
        'iu'
      );
    }
    const whitelistRe = rule.whitelist?.length
      ? rule.whitelist.map(w => new RegExp(w.replace(/[.*+?^${}()|[\]\\]/g,'\\$&'), 'i')) : [];
    return { rule, regex, whitelistRe };
  }

  /** 返回命中的规则（可多条），未命中返回 null */
  match(text: string, username: string, ctx: RuleScope): Rule[] {
    const cleanText = sanitize(text);      // 去零宽字符
    const cleanUser = sanitize(username).replace(/[\s_.-]+/g, '');  // 用户名额外去标点
    const hits: Rule[] = [];

    for (const c of this.compiled) {
      if (c.rule.scope !== 'all' && c.rule.scope !== ctx) continue;

      const target = c.rule.type === 'username' ? cleanUser : cleanText;
      if (!c.regex.test(target)) continue;

      // 白名单例外
      if (c.whitelistRe.some(re => re.test(cleanText))) continue;

      hits.push(c.rule);
    }
    return hits;
  }
}
```

```ts
// lib/rules/sanitize.ts —— 反绕过清洗（x-comment-blocker 精华）
const INVISIBLE_CHARS = /[\u00AD\u180E\u200B-\u200F\u202A-\u202E\u2060-\u206F\uFEFF]/g;
const SPAM_CHARS = /[\u0300-\u036F\u1AB0-\u1AFF\u1DC0-\u1DFF\u20D0-\u20FF\uFE20-\uFE2F]/g; // 组合附加符号

export function sanitize(s: string): string {
  return s
    .replace(INVISIBLE_CHARS, '')    // 去零宽字符（防 "免\u200b费"）
    .replace(SPAM_CHARS, '')         // 去花体字修饰符
    .normalize('NFKC');              // 全角→半角（防 "１２３" "ｆｒｅｅ"）
}
```

> **为什么 wordBoundary 默认**：调研发现 xModerator 的 `sex` 命中 "Sussex"、`cum` 命中 "document"、`rape` 命中 "scrape"、`ad` 匹配一切。这是子串匹配的致命伤。**默认词边界 + 中文 CJK 边界**能消除绝大多数误伤，用户可对特定词降级为 substring。

### 4.5 折叠占位卡 UI（ActionExecutor + Shadow DOM）—— 咱们的最大差异化

**命中后不直接删除，替换为可展开的占位卡**：

```ts
// lib/action.ts
export class ActionExecutor {
  /** 折叠一条推文为占位卡 */
  fold(cell: HTMLElement, hits: Rule[], group: string) {
    if (cell.dataset.folded) { this.addToGroup(cell, group); return; }
    cell.dataset.folded = '1';

    // 保留原始内容引用，便于还原
    const original = cell.firstElementChild as HTMLElement;
    original.style.display = 'none';

    // 用 Shadow DOM 注入占位卡（隔离样式，不被 X CSS 污染）
    const host = document.createElement('div');
    host.className = 'tfc-fold-host';
    const sr = host.attachShadow({ mode: 'open' });
    sr.innerHTML = this.renderPlaceholder(hits, group);
    cell.insertBefore(host, original);

    // 点击展开
    sr.querySelector('[data-action="expand"]')!.addEventListener('click', () => {
      this.unfold(cell);
    });
  }

  unfold(cell: HTMLElement) {
    const host = cell.querySelector('.tfc-fold-host');
    (cell.firstElementChild as HTMLElement).style.display = '';
    host?.remove();
    delete cell.dataset.folded;
  }

  private renderPlaceholder(hits: Rule[], group: string): string {
    const reason = hits[0]?.category ?? 'spam';
    return `
      <style>${PLACEHOLDER_CSS}</style>
      <div class="tfc-card" role="note">
        <span class="tfc-icon">🛡️</span>
        <span class="tfc-text">已屏蔽内容（疑似 ${reason}）</span>
        <button class="tfc-btn" data-action="expand">展开查看</button>
      </div>
    `;
  }
}
```

**聚合展示**（可选增强）：连续多条命中可聚合成"已屏蔽 N 条 [全部展开]"，减少视觉噪音。

### 4.6 缓存（CacheManager）—— 性能关键

```ts
// lib/cache.ts —— x-comment-blocker 的 __cbxQuickHash 思路
let filterVersion = 0;   // 规则/设置一变就自增，让旧 hash 失效

export const cacheManager = {
  clear() { /* 清所有节点缓存 */ },

  shouldReprocess(cell: HTMLElement, text: string, handle: string): boolean {
    const hash = `${text}|${handle}|${filterVersion}`;
    if (cell.dataset.tfcHash === hash) return false;   // 未变，跳过
    cell.dataset.tfcHash = hash;
    return true;
  },

  bumpVersion() { filterVersion++; this.clear(); },
};
```

**另外维护去重计数 Set**（key = 归一化文本 + handle），带 TTL 30min + 容量上限 5000，防内存膨胀。

### 4.7 词库同步（Background + GitHub CDN）

```ts
// scripts/sync-keywords.ts —— x-comment-blocker 的 ETag 方案
const KEYWORDS_URL = 'https://api.github.com/repos/<你的repo>/contents/keywords.txt';
const SYNC_INTERVAL_MIN = 360;   // 6 小时

export async function syncKeywords() {
  const { cloudETag, cloudKeywords } = await storage.get(['cloudETag', 'cloudKeywords']);
  const res = await fetch(KEYWORDS_URL, {
    headers: {
      'Accept': 'application/vnd.github.v3.raw',
      ...(cloudETag ? { 'If-None-Match': cloudETag } : {}),
    },
  });
  if (res.status === 304) return;           // 未变，省配额
  if (res.status === 403 || res.status === 429) {
    storage.set({ syncError: 'API 限流' });
    return;
  }
  const text = await res.text();
  await storage.set({
    cloudKeywords: text,
    cloudETag: res.headers.get('ETag') ?? '',
    lastSyncTime: Date.now(),
    syncError: '',
  });
  notifyContentScript('rules-updated');     // 通知 content script 重载规则
}
```

**优势**：零后端、改 `keywords.txt` 推 GitHub 即可热更新、ETag 省 API 配额、用户可 fork 自定义。

### 4.8 可选 LLM 精判（Roadmap，opt-in）

关键词预筛命中后，**可选**调 LLM 二次确认（降低误伤）。架构参考 twitter-blocker：

```
关键词命中(本地) → [若开启 LLM 精判] → 用户自建 Worker / 用户自填 key
                                     → 返回 {isSpam, confidence, reason}
                                     → confidence 高 → 折叠
                                     → confidence 低 → 放行（宁错放不误伤）
```

**隐私设计**：默认关闭；开启时明确告知数据会上传；支持用户自填 OpenAI key（本地存，不上传到咱们服务器）或自部署 Worker。

### 4.9 存储设计（避开 xModerator 的 storage key bug）

```ts
// lib/storage.ts —— 统一 key 命名，杜绝前后端不一致
const STORAGE_KEYS = {
  SETTINGS: 'tfc:settings',        // sync（跨设备，<100KB）
  RULES:    'tfc:rules',           // local（用户规则 + 内置规则合并）
  CLOUD:    'tfc:cloudKeywords',   // local（云端词库）
  STATS:    'tfc:stats',           // local（统计）
```

## 五、关键流程：一条评论从出现到被过滤

```
1. X 渲染新评论 → DOM 新增 cellInnerDiv
2. ObserverManager 收集到 pending Set，100ms 后 flush
3. DomExtractor.extractTweet(cell) → { text, username, handle, hasMedia }
4. CacheManager.shouldReprocess? → 否则跳过
5. ContextDetector: 当前页是 status? 是主推文? → 决定 scope
6. RuleEngine.match(text, username, scope)
     - sanitize 清洗（去零宽/花体/全角）
     - 遍历 compiled rules（按 scope 过滤）
     - 词边界正则匹配 + 白名单例外
   → 命中返回 hits[]
7. 若 hits 非空:
     - [可选] LLM 精判确认
     - ActionExecutor.fold(cell, hits) → Shadow DOM 占位卡
     - 更新 stats + history
     - 加入去重 Set
8. 若无命中: 确保无 folded 状态（处理规则变更后还原）
```

## 六、误伤防护（多道防线）

| 防线 | 实现 |
|---|---|
| 1. 词边界默认 | 规则默认 `wordBoundary`，中文用 CJK 边界 |
| 2. 白名单例外 | 每条规则可配 whitelist |
| 3. 折叠不删除 | 命中只折叠，一键展开，**永不破坏性删除** |
| 4. 主推文豁免 | 永远放行主推文，只过滤回复 |
| 5. scope 控制 | 规则可限定 comment/feed/all |
| 6. dry-run 模式 | 设置项：只标记不折叠，观察一段时间再启用 |
| 7. 全局白名单用户 | 指定用户名永不过滤 |
| 8. 历史可追溯 | 命中记录含 user/text/reason，可申诉复核 |
| 9. 一键全展开 | 紧急情况下展开所有折叠内容 |

## 七、抗 X 改版策略

借鉴 Control Panel for Twitter 的多重策略：

1. **data-testid 优先**，几乎不用 className
2. **多 selector 兜底链**（每个锚点配 2-3 个 fallback）
3. **selector 集中管理**（一个 `selectors.ts`，改版时改一处）
4. **selector 存活性自检**（启动时检测关键 selector 是否存在，失效则降级 + 上报）
5. **详尽日志**（debug 模式打印每步，便于排障）
6. **可选 main world 注入**（未来读 React state 用，最强抗改版）
